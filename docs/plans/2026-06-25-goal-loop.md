# Northstar Superpowers `goal-loop` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fork `obra/superpowers` into a public MIT-licensed set named **Northstar Superpowers**, adding one combined skill `goal-loop` that defines an airtight goal through adversarial interrogation (hard-gated) and then drives an autonomous quality-convergence loop until the goal is met and nothing is provably broken.

**Architecture:** A clean rebrand of the upstream plugin (manifests, attribution, session-start hook namespace), plus a new `skills/goal-loop/` skill with two phases (Define, Run) reached through two thin `commands/` shims (`/goal`, `/loop`). The skill *orchestrates over* existing skills (TDD, systematic-debugging, verification-before-completion, `/simplify`, `/code-review`) rather than reimplementing them. Skills are authored TDD-style: every behavioral claim is proven with a subagent pressure scenario (RED baseline → GREEN with skill).

**Tech Stack:** Markdown skills + JSON plugin manifests (Claude Code plugin format); bash session-start hook; `gh` CLI for the public repo. No application runtime.

## Global Constraints

- **License:** Keep upstream `LICENSE` verbatim (MIT, © 2025 Jesse Vincent). Add a second copyright line for the fork below it; never remove the original. — copied from spec §2.
- **Attribution:** `README` and `plugin.json` `description` must state "Forked from obra/superpowers." A `NOTICE` file links upstream. — spec §2.
- **Plugin name:** `northstar-superpowers` everywhere a name appears (plugin.json, marketplace.json, hook injection). — spec §2.
- **No stop-caps:** The Run phase has no iteration or token cap in the skill body. Stop condition is *goal met + nothing provably broken* only. — spec §6.2.
- **Surgical dead-code rule:** Cleanup removes only orphans the loop itself created; pre-existing dead code is **flagged, not deleted**, unless `GOAL.md` authorizes it. — spec §6.3.
- **Skill authoring is TDD-via-subagents:** No skill section ships without a pressure scenario where a subagent fails the baseline and complies with the skill. — writing-skills.
- **No Co-Authored-By trailers** in commit messages (user standing preference).

- [ ] **Step 1: Write the LOOP-LOG template**

- [ ] **Step 2: Write the Run-phase reference**

Create `skills/goal-loop/references/run-phase.md` documenting, per spec §6:
- **Verification surfaces:** static/test (tests, types, lint → on failure use test-driven-development + systematic-debugging); runtime/browser via `preview_*` (`preview_start` → reload → `preview_console_logs`/`preview_network` → `preview_snapshot` → `preview_screenshot`/`preview_click`/`preview_fill`); a surface that can't exercise the change is **skipped, not failed**.
- **Iteration:** re-verify GOAL.md criteria with their named checks (reuse verification-before-completion — evidence, never assertions) → if all pass and no surface reports an issue, run convergence passes → if clean, STOP and report → else fix next issue via TDD/systematic-debugging, commit, repeat.
- **Convergence passes:** dead-code/cleanup (orphans the loop created; **flag, don't delete** pre-existing dead code unless GOAL.md authorizes); simplify/reuse (`/simplify` + `/code-review`); append LOOP-LOG; efficient re-verification (only re-check what the last change could affect).
- **Coding discipline:** reuse-before-writing (search via Explore first), surgical changes, YAGNI.
- **Stop condition:** goal met + nothing provably broken. No iteration/budget cap.
